Project Context:
  • Objective: Assess effort required for customer data cleanse/harmonization
  • Stakeholders: Stephen (Data Architect), Silamir, Dipesh, Reem, Group IT
  • Target Platform: Snowflake (migrating from Azure Databricks)

Assessment Framework

Phase 1: Discovery & Analysis

  • 1.1 Source System Inventory
    • Document all customer data sources requiring harmonization
    • Identify data formats, volumes, and update frequencies
    • Map current data flows and dependencies
    Output: Source Systems Catalog
  • 1.2 Current State Assessment
    • Analyze existing data quality issues
    • Document current ETL processes in Azure Databricks
    • Identify technical debt and pain points
    Output: Current State Analysis

Phase 2: Target Design

  • 2.1 Data Quality Rules Definition
    • Define customer data quality standards
    • Create validation rules and thresholds
    • Establish data governance policies
    Output: Data Quality Framework
  • 2.2 Target Schema Design
    • Design unified customer schema for CRM processes
    • Define data model for Snowflake
    • Create mapping specifications
    Output: Target Schema Documentation

Phase 3: Solution Architecture

  • 3.1 ETL/ELT Design
    • Document data cleansing processes
    • Design harmonization workflows
    • Define interim solution architecture
    Output: ETL Architecture
  • 3.2 CRM Integration Design
    • Define integration patterns with CRM
    • Specify data refresh frequencies
    • Document API/interface requirements
    Output: Integration Specifications

Phase 4: Effort Estimation

  • 4.1 Implementation Roadmap
    • Break down work into sprints/phases
    • Identify resource requirements
    • Define milestones and dependencies
    Output: Implementation Roadmap
  • 4.2 Effort Estimate
    • Provide detailed effort breakdown
    • Include build, test, and deployment estimates
    • Factor in operational support needs
    Output: Effort Estimation Document
